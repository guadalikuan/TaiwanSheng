# 部署检查清单

在按下"发布"按钮之前，请务必逐项确认。任何一个疏漏都可能导致问题。

## 第一阶段：伪装与基础设施 (Cover & Infra)

### 域名和服务器

- [ ] **域名注册**：切勿使用阿里云/腾讯云。使用 Namecheap 或 GoDaddy，开启 Whois 隐私保护。建议后缀 `.xyz` 或 `.network`（例如：`tws-protocol.xyz`）
- [ ] **服务器节点**：选择**新加坡**或**日本**的 AWS/DigitalOcean 节点。速度要快，距离台湾要近
- [ ] **SSL 证书**：必须是 HTTPS。浏览器地址栏的小锁是信任的第一步
- [ ] **DNS 配置**：确保域名正确解析到服务器 IP

### 通信配置

- [ ] **Telegram Bot**：申请一个新的 Bot Token，名称不要带"China"，建议用英文代号（如 `TWS_Sentinel_Bot`）
- [ ] **Telegram 频道**：建立官方频道，预先填充内容

## 第二阶段：资金与资产 (Money & Assets)

### 区块链配置

- [ ] **智能合约部署**：在 BSC (Binance Smart Chain) 部署代币合约。BSC 手续费低，且在东南亚/台湾渗透率高
- [ ] **合约地址配置**：在 `.env.production` 中设置 `VITE_CONTRACT_ADDRESS`
- [ ] **收款钱包**：创建一个多签钱包 (Gnosis Safe)，不要用个人钱包直接收款

### 资产数据

- [ ] **资产数据录入**：将第一批"工抵房"数据（脱敏处理，只留城市、地段、面积、原价）录入数据库
- [ ] **数据验证**：确保所有资产数据格式正确

## 第三阶段：心理战准备 (Psy-Ops Prep)

### 种子数据

- [ ] **种子数据注入**：后台手动生成 500 个"虚假交易记录"和"活跃特工"。不能让第一批进来的用户看到空荡荡的房间
- [ ] **数据真实性**：确保种子数据看起来真实可信

### 倒计时配置

- [ ] **倒计时校准**：将首页的倒计时设置为一个模糊的、具有压迫感的日期（例如：距离"2027年XX月XX日"或纯粹的动态算法）
- [ ] **倒计时测试**：确保倒计时在不同时区正确显示

### 内容准备

- [ ] **Telegram 频道内容**：预先填充 20 条英文+繁体中文混合的"快讯"，内容多为台海局势紧张的新闻摘要
- [ ] **内容审核**：确保所有内容符合要求

## 第四阶段：法律防火墙 (Legal Firewall)

### 法律文件

- [ ] **协议打印**：那份给大陆老板签的《资产数字化托管协议》，打印出来，盖上公司公章
- [ ] **协议存档**：确保所有协议文件妥善保存

### 免责声明

- [ ] **免责声明添加**：在 APP 的 Footer 添加极小的文字：*"This is a gamified asset simulation. Not financial advice."*
- [ ] **免责声明可见性**：确保免责声明在所有页面可见（虽然字体可以很小）

## 技术检查

### 构建和部署

- [ ] **构建测试**：本地运行 `npm run build:prod` 确保构建成功
- [ ] **环境变量**：确认所有生产环境变量已正确配置
- [ ] **Docker 测试**：如使用 Docker，确保镜像构建和运行正常
- [ ] **Nginx 配置**：确认 Nginx 配置正确，SPA 路由正常工作

### 功能测试

- [ ] **路由测试**：测试所有页面路由是否正常
- [ ] **API 连接**：测试 API 连接是否正常
- [ ] **区块链连接**：测试区块链连接是否正常
- [ ] **移动端适配**：测试移动端显示是否正常

### 性能和安全

- [ ] **性能测试**：测试页面加载速度
- [ ] **安全头配置**：确认安全头已正确配置
- [ ] **HTTPS 配置**：确认 HTTPS 正常工作
- [ ] **错误处理**：测试错误处理是否正常

## 监控和备份

- [ ] **监控配置**：配置应用监控和日志
- [ ] **备份策略**：制定数据备份策略
- [ ] **健康检查**：配置健康检查端点

- [ ] **团队审核**：团队内部审核通过
- [ ] **应急预案**：制定应急预案
