# 资产市场页面说明

## 📋 页面概述

点击"资产"按钮后进入的是**资产市场页面**（`/market`），这是一个公开的市场展示页面，使用**军事游戏化风格**展示所有已审核通过并上架的房产资产。

---

## 🎯 页面功能

### 核心目的
展示所有**已审核通过**的房产资产，供用户浏览和购买。这些资产已经过审核确权，状态为 `AVAILABLE`（可交易）。

---

## 🖼️ 页面布局说明

### 1. **顶部状态栏**
```
[LINK: ENCRYPTED]                    [时间] // ZONE: UTC+8
```
- **LINK: ENCRYPTED**：显示加密连接状态（营造军事感）
- **时间**：当前服务器时间（UTC+8，北京时间）

---

### 2. **随机分配区域（The Gacha）**

这是页面的核心功能区域：

**标题**：`RANDOM ALLOCATION`（随机分配）

**说明**：
- 系统将根据当前的"战争游戏模拟"自动分配最优的避难所
- 警告：检测到台北节点的高需求

**概率显示**：
- S-TIER（S级）：2%
- A-TIER（A级）：18%
- B-TIER（B级）：80%

**按钮**：`SECURE NOW` - 100 USDT
- 点击后随机分配一个资产
- 这是一个**盲盒机制**（Gacha），用户付费后随机获得资产

---

### 3. **筛选栏**

可以按战区筛选资产：
- **ALL**：显示所有资产
- **NORTH-WEST**：西北战区
- **SOUTH-WEST**：西南战区
- **CENTRAL**：中央战区
- **COASTAL(RISKY)**：沿海战区（高风险）

---

### 4. **资产卡片列表**

每个资产卡片显示以下信息：

#### 卡片头部
- **UNIT ID**：资产唯一标识符（如：`CN-XI-BKR-4921`）
- **SECTOR**：战区标识（如：`SECTOR-CN-NW (西北战区)`）
- **STATUS**：状态标签
  - `AVAILABLE`：可用（绿色）
  - `ALMOST GONE`：几乎售罄（红色闪烁）
  - `CRITICAL`：紧急状态（红色闪烁）

#### 资产属性（伪装成军事参数）

这些是将房产属性伪装成军事装备参数：

| 显示标签 | 实际含义 | 说明 |
|---------|---------|------|
| **CLASS** | 资产类型 | 如：`BUNKER (加固地堡)`、`CMD (指挥所)`<br/>对应房产的面积大小和类型 |
| **STRAT DEPTH** | 战略深度等级 | 如：`LV.3 (INLAND)`<br/>表示资产的内陆深度：<br/>- DEEP：深内陆（4-5级）<br/>- INLAND：内陆（2-3级）<br/>- COASTAL：沿海（1级） |
| **DEFENSE RATING** | 防御等级 | 如：`MED`、`HIGH`、`MAX`<br/>基于安全指数（1-5星）：<br/>- 5星 → MAX<br/>- 4星 → HIGH<br/>- 3星 → MED<br/>- 2星 → LOW<br/>- 1星 → MIN |

#### 剩余容量进度条
```
REMAINING CAPACITY: XX%
[红色进度条]
```
- 显示资产的剩余可购买份额百分比
- 进度条越短，说明越稀缺
- 用于营造紧迫感和稀缺性

#### 价格
```
XXXX TWS
```
- 价格以 TWS Token 计价
- 数值基于资产的 Token 总数除以 100

#### 操作按钮
- 点击十字准星图标（`Crosshair`）可以查看资产详情

## 🔍 数据含义详解

### 战区（Sector）

将房产所在城市映射为军事战区：

| 战区代码 | 含义 | 对应区域 |
|---------|------|---------|
| `CN-NW` | 西北战区 | 西北地区城市（如西安、咸阳） |
| `CN-QINLING` | 秦岭战区 | 秦岭附近城市（如宝鸡、商洛） |
| `CN-INT` | 内陆储备区 | 其他内陆城市 |
| `CN-IND` | 工业区 | 工业城市 |

### 资产类型（Type/CLASS）

根据房产面积和价格自动分类：

- **BUNKER (加固地堡)**：小户型（≤120㎡）
- **CMD (指挥所)**：大户型（>120㎡）

### 防御等级（Defense Rating）

基于房产的地理位置安全系数计算：

- **位置系数高**（≥1.8）：深内陆 → 安全系数高 → 防御等级 MAX
- **位置系数中**（1.0-1.8）：内陆 → 安全系数中 → 防御等级 MED-HIGH
- **位置系数低**（<1.0）：沿海 → 安全系数低 → 防御等级 LOW-MIN

### 战略深度（Strategic Depth）

表示资产距离海岸线的战略纵深：

- **DEEP**：深度内陆（位置系数≥1.5）
- **INLAND**：内陆（位置系数1.0-1.5）
- **COASTAL**：沿海（位置系数<1.0）

**含义**：距离海岸线越远，越安全（避免海上打击）

## 🎮 游戏化设计说明

### 为什么使用军事风格？

这是一个**心理设计**，将房产投资包装成"战争避险资产"：

1. **营造紧迫感**：通过"战区"、"防御等级"等军事术语，让用户感受到"战争即将到来"的紧迫性

2. **增加价值感**：将普通房产包装成"战略储备资产"，提升心理价值

3. **降低抵触情绪**：通过游戏化包装，让购买房产更像是在"玩游戏"而不是"投资"

4. **目标用户心理**：针对台湾用户，使用军事术语符合其对"统一"的预期心理

### 设计元素

- **暗色背景 + 网格线**：营造军事指挥室氛围
- **红色警告元素**：营造危机感
- **专业军事术语**：增加真实感和专业性
- **实时数据更新**：营造市场活跃度

## 📊 数据来源

### 显示的资产

- **来源**：`GET /api/arsenal/assets` API
- **筛选条件**：只显示状态为 `AVAILABLE`（已审核通过）的资产
- **数据格式**：脱敏后的资产数据（不包含敏感信息）

### 数据转换

原始脱敏数据 → 市场格式：
- `locationTag` → `SECTOR`（战区）
- `securityLevel` → `DEFENSE RATING`（防御等级）
- `securityLevel` → `STRAT DEPTH`（战略深度）
- `specs.type` → `CLASS`（类型）
- `financials.totalTokens` → `price`（价格）

## ⚠️ 注意事项

1. **显示的是脱敏数据**：
   - 不显示真实地址、门牌号
   - 只显示代号、战区、类型等公开信息

2. **数据是实时的**：
   - 资产列表会实时更新
   - 已售出的资产会从列表中移除

3. **价格以 TWS Token 计价**：
   - 不是人民币价格
   - 需要兑换 TWS Token 才能购买
